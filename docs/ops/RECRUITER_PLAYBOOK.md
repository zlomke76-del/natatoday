NATA Today — Internal Operations Playbook
Recruiter / Agent Execution Guide
🎯 Mission

Operate a controlled, high-signal hiring pipeline where:

No candidate reaches a dealer without being interview-ready, owned, and fully contextualized.

This is not a volume system.
This is a quality + clarity + readiness system.

🧭 System Roles
Lead Recruiter
Conducts virtual interviews
Sets quality standard
Finalizes interview packets
Determines dealer readiness
Recruiters
Conduct interviews (as capacity allows)
Approve / hold / pass candidates
Own assigned candidates
Ensure progression to dealer-ready state
Agents
Source candidates
Manage follow-ups
Prepare candidates for interviews
Maintain pipeline movement
🧠 Core System Principles
1. State Drives Progression
If the state is insufficient → the system blocks progression.

No manual overrides. No assumptions.

2. Assignment Drives Ownership
Every candidate must have a clear owner.

Ownership determines:

who schedules interviews
who follows up
who ensures completion
who is accountable
3. Dealers See Only Decision-Ready Candidates

Dealers never see:

raw applicants
unreviewed candidates
candidates without packets
4. Execution Must Be Controlled
Movement only occurs when all required conditions are satisfied.
🔄 Candidate Lifecycle (Operational)
new
→ screening
→ assignment
→ virtual_invited
→ virtual_scheduled
→ virtual_completed
→ packet_ready
→ dealer_interview_scheduled
→ dealer_review
→ placed | not_hired | keep_warm
🧩 ASSIGNMENT CONTROL (CRITICAL LAYER)
Ownership Rules
Dealer-assigned recruiter → ALWAYS owns
Manual admin assignment → ALWAYS owns
Existing assignment → MUST be respected
Auto-assignment → ONLY under imbalance conditions
Unassigned Candidates
Unassigned candidates are system risk.
They must be surfaced and resolved quickly.
Assignment Responsibility

The assigned recruiter is responsible for:

Scheduling interview
Ensuring attendance
Capturing notes
Completing packet
Advancing candidate correctly

No shared responsibility. No ambiguity.

⚠️ INTERVIEW CAPACITY CONTROL

Interviews are the primary system constraint.

Rule:
If interview capacity is exceeded:
→ assignment must redirect overflow
OR
→ scheduling must be throttled
System Behavior Under Load

When backlog increases:

System shifts from manual → assist → controlled auto

Auto-assignment is:

A pressure valve, not a default behavior
🧩 STAGE-BY-STAGE EXECUTION
1. Candidate Intake (Agent)

Status: new → screening

Responsibilities:
Validate contact information
Confirm basic role alignment
Identify early availability signals
Outcome:
Reject → not_fit
Advance → assignment
2. Assignment

Status: assignment

Must Determine:
Who owns this candidate?
Possible Sources:
Dealer default recruiter
Admin assignment
System-assisted assignment
3. Virtual Interview Scheduling (Agent / Recruiter)

Status: virtual_invited

Actions:
Send scheduling link
Confirm interview time
Ensure candidate readiness
Goal:
Candidate arrives prepared and on time
4. Virtual Interview (Recruiter)

Status: virtual_completed

Evaluate:
Communication
Clarity
Responsiveness
Presence
Confidence
Professionalism
Fit
Role understanding
Expectations
Availability
Schedule reliability
Weekend readiness
Required Output:
Clear, structured, decision-driving notes

No vague feedback.

5. Interview Packet Generation

Status: packet_ready

Packet MUST Include:
Evaluation
Strengths
Concerns
Recommendation
Interview Questions
Targeted to candidate gaps
Verification Items
Dealer must confirm
Resume
Always attached
Quality Gate
If packet is weak → DO NOT ADVANCE
6. Dealer Interview Scheduling

Status: dealer_interview_scheduled

Requirements:
✔ Packet complete
✔ Resume attached
✔ Candidate vetted
✔ Availability confirmed
7. Dealer Board Transition
Recruiter control ENDS
Dealer control BEGINS

Dealer sees:

Candidate
Packet
Interview time
8. Dealer Decision

Options:

Hired
Not hired
Keep warm
No-show
Needs follow-up

With:

Reason required
9. Outcome Handling
If Hired:
→ mark placed
→ close job
→ update system
If Not Hired:
→ retain candidate
→ eligible for future roles
If Keep Warm:
→ high-value pool
→ future outreach candidate
🔁 CANDIDATE REUSE SYSTEM

Candidates are persistent assets.

Process:
New job opens
→ match against existing candidates
→ initiate outreach
Outreach Rules

Only reach out if:

Fit ≥ previous fit
AND availability still valid
