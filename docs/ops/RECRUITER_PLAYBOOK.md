# NATA Today — Internal Operations Playbook

### Recruiter / Agent Execution Guide (Don + Team)

---

## 🎯 Mission

Operate a **controlled, high-signal hiring pipeline** where:

```txt
No candidate reaches a dealer without being interview-ready and fully contextualized.
```

Your job is not volume.
Your job is **quality + clarity + readiness**.

---

## 🧭 System Roles

### Don (Lead Recruiter)

* Conducts virtual interviews
* Sets quality bar
* Finalizes interview packets
* Decides when a candidate is dealer-ready

### Agents (Recruiters)

* Source candidates
* Manage follow-ups
* Prep candidates for virtual interview
* Maintain pipeline movement

---

## 🔄 Candidate Lifecycle (Operational)

```txt
new
→ screening
→ virtual_invited
→ virtual_completed
→ packet_ready
→ dealer_interview_scheduled
→ dealer_review
→ placed | not_hired | keep_warm
```

---

## 📌 Core Rule (Non-Negotiable)

```txt
If the state is insufficient → the candidate does not move forward.
```

No exceptions. No “close enough.”

---

# 🧩 STAGE-BY-STAGE EXECUTION

---

## 1. Candidate Intake (Agent)

**Status:** `new` → `screening`

### Responsibilities:

* Review application
* Confirm:

  * Contact info valid
  * Basic role alignment
  * Availability (early signal)

### Outcome:

```txt
Reject → not_fit
Advance → virtual_invited
```

---

## 2. Virtual Interview Scheduling (Agent)

**Status:** `virtual_invited`

### Actions:

* Send interview link (NATA studio)
* Confirm time
* Ensure candidate shows

### Goal:

```txt
Candidate arrives prepared and on time
```

---

## 3. Virtual Interview (Don)

**Status:** `virtual_completed`

Conducted inside:

```txt
/recruiter/interviews/[applicationId]
```

---

### During Interview — Evaluate:

#### 1. Communication

* Can they carry a conversation?
* Can they explain clearly?

#### 2. Presence

* Confidence level
* Professional tone

#### 3. Fit

* Role understanding
* Realistic expectations

#### 4. Availability

* Schedule reliability
* Weekend readiness (critical)

---

### Capture Notes (Required)

```txt
Clear, structured, decision-driving notes only.
```

Avoid fluff.

---

## 4. Interview Packet Generation (System + Don)

Triggered after notes saved.

**Status:** `packet_ready`

---

### Packet MUST include:

#### NATA Notes (Required)

```txt
Clear evaluation:
- strengths
- concerns
- recommendation
```

#### Interview Questions (Auto-guided)

```txt
Specific to candidate gaps
```

#### Verification Items

```txt
What the dealer MUST confirm
```

#### Resume

```txt
Always attached or generated
```

---

### Quality Gate

```txt
If packet is weak → DO NOT ADVANCE
```

---

## 5. Dealer Interview Scheduling

**Status:** `dealer_interview_scheduled`

### Requirements BEFORE scheduling:

```txt
✔ Packet complete
✔ Resume attached
✔ Candidate vetted
✔ Availability confirmed
```

Then:

```txt
Schedule → set dealer_interview_at
```

---

## 6. Dealer Board (Hands-Off Moment)

At this point:

```txt
Recruiter control ENDS
Dealer control BEGINS
```

Dealer sees:

* Candidate
* Packet
* Interview time

---

## 7. Dealer Decision

Dealer submits:

```txt
Hired
Not hired
Keep warm
No-show
Needs follow-up
```

With:

```txt
WHY (required)
```

---

## 8. Outcome Handling (Agent)

### If Hired:

```txt
→ mark placed
→ job closes
→ pipeline updated
```

---

### If Not Hired:

```txt
→ candidate retained
→ tagged for future roles
→ remains in system
```

---

### If Keep Warm:

```txt
→ high-value candidate pool
→ eligible for outreach
```

---

# 🔁 CANDIDATE REUSE SYSTEM

All candidates remain assets.

Agents must:

```txt
Monitor new job openings
→ match existing candidates
→ proactively reach out
```

---

## 🎯 Outreach Rules

Only reach out if:

```txt
Fit ≥ previous role fit
AND availability still valid
```

---

### Agents:

```txt
Review new applicants
Schedule virtual interviews
Follow up no-shows
Prepare candidates
```

---

### Don:

```txt
Run interviews

```

# 🚨 FAILURE MODES (WATCH THESE)

### ❌ Pushing candidates too early

Fix:

```txt
Enforce packet_ready = true
```

---

### ❌ Weak notes

Fix:

```txt
Notes must drive a hiring decision
```

---

### ❌ Dealer sees noise

Fix:

```txt
Strict visibility gating
```

---

### ❌ Candidate ghosting

Fix:

```txt
Agent follow-up discipline
```
