# NATA Today — Hiring Operating System

## Overview

NATA Today is a **dealer-first hiring system** that replaces fragmented recruiting workflows with a structured, end-to-end pipeline:

```txt
Dealer request → Job generated → Candidate applies → Screening → Virtual interview → Packet → Dealer interview → Decision → Placement
```

The system is built to:

* Reduce dealer workload
* Improve candidate quality
* Control hiring flow from intake to placement
* Enable scalable recruiter + agent operations

---

## Core Architecture

### 1. Dealer Layer (What the dealer sees)

* Only **ready-to-interview candidates**
* No noise, no early-stage applicants
* Decision-focused UI:

```txt
Interview → Approve / Reject / Keep Warm → Document why
```

---

### 2. Recruiter / Don Control Center

Operational cockpit for:

* Virtual interviews (Daily embedded studio)
* Notes capture
* Packet generation
* Scheduling dealer interviews

Flow:

```txt
virtual_invited
→ interview conducted
→ notes saved
→ packet generated
→ dealer interview scheduled
```

---

### 3. Agent Portal (future-ready)

* Recruiters assigned to candidates/dealers
* Performance tracking
* Candidate sourcing + outreach

---

## Candidate Lifecycle

```txt
new
→ screening
→ virtual_invited
→ virtual_completed
→ packet_ready
→ dealer_interview_scheduled
→ dealer_review
→ placed | not_hired | keep_warm
```

---

## Key System Rules

### Visibility Control

Candidates ONLY appear on dealer dashboard if:

```txt
status = dealer_interview_scheduled
AND dealer_interview_at IS NOT NULL
AND interview_packet_ready = true
```

---

### No Manual Overrides

* No hardcoded candidates
* No manual SQL for interview questions
* No dealer exposure before readiness

Everything must be:

```txt
state-driven
```

## Interview Studio (Daily Integration)

We use Daily to power embedded video inside NATA.

### Flow:

```txt
POST /api/nata/interviews/room
→ creates Daily room

Recruiter opens:
→ /recruiter/interviews/[applicationId]

Interview happens inside NATA UI
→ notes captured
→ saved to Supabase
```

## Interview Packet System

Generated AFTER virtual interview.

Stored in:

```txt
nata.interview_packets
```

Includes:

* NATA / Solace notes
* Resume link
* Interview questions (auto-generated)
* Verification items

## System Philosophy

This is NOT a job board.

This is:

```txt
A controlled hiring pipeline
```

Principles:

* Dealers don’t create jobs → Solace does
* Candidates are filtered BEFORE dealer interaction
* Every decision must be documented
* No candidate reaches a manager without sufficient state
